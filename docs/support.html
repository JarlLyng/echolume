FAQ

Echolume needs microphone permission — why?
On macOS, access to audio inputs (including audio interfaces) is granted via the same "microphone" permission. Echolume only uses this to read the audio stream for the visuals. It does not record or send audio anywhere.
No sound / no signal?
Check that you've selected the correct Audio Input Device and, if applicable, the right channel pair. Ensure the input is not muted and that the level is sufficient. If you use an audio interface, make sure it's connected and chosen in Echolume. See the setup guide for details.
Can I use system audio (e.g. Spotify) as input?
Echolume does not capture "system output" directly. You can use a loopback or virtual device (e.g. BlackHole, Soundflower) that routes system audio into an input, then select that input in Echolume.
Which macOS version is required?
Echolume requires macOS 14 (Sonoma) or later.
